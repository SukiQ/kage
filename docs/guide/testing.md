# 质量测试

AI 规划缺失的测试用例，生成测试代码并写入项目，可直接运行验证。

## 功能

- **测试规划**：AI 分析代码与现有测试，估算覆盖率、列出高风险模块、推荐缺失用例。
- **推荐用例**：每条含模块、优先级、类别（边界值 / 异常流程 / 状态转换 …）、测试类型、描述与重要性。
- **生成测试代码**：为指定用例生成测试代码，**由 AI 直接写入项目正确的测试路径**（如 Java 的 `src/test/java/...XxxTest.java`、Dart 的 `test/*_test.dart`），并在界面预览。
- **运行测试**：单用例或整项目运行，AI 自动选择构建工具命令（`mvn test` / `gradlew test` / `flutter test` / `pytest` …）并解读结果。
- **失败解读**：测试失败时 AI 给出原因分析。
- **导出计划**：一键导出测试计划 Markdown。

## 使用步骤

1. 进入「质量测试」，点「AI 规划测试」。
2. 在推荐用例列表中，对某条点「生成测试代码」——代码会写入项目并显示「已写入」。
3. 点「运行」执行该测试，查看通过 / 失败结果；失败时查看 AI 解读。
4. 或点「执行测试」运行项目全部测试。

::: tip 语言无关
测试代码的路径与运行命令均由 AI 按项目实际决定，Kage 不做语言硬编码，Java / Python / Dart / 前端等均可。
:::
